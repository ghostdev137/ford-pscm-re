# Transit — LKA Full Authority + 2× Driver Override Tolerance

**File:** `LKA_FULL_AUTHORITY_OVERRIDE_2X.VBF` (66,915 bytes)
**Vehicle:** 2025 Ford Transit
**Base cal:** `LK41-14D007-AH`
**Flash address:** `0x00FD0000`
**Header checksum:** `0x26F3D993` (zlib CRC32, recomputed — verified)

## Goal

Combine every previously drive-confirmed Transit LKA patch with a new
**driver-override torque threshold** bump targeting the "can't make big
turns" symptom. Addresses a separate failure mode from torque authority
and lockout timers — namely, the rack yielding assist mid-turn when
reaction torque on the steering sensor crosses the quiet-gate threshold.

See `analysis/transit/driver_override_patch_candidates.md` for the evidence
chain that identifies `cal+0x29D4`/`cal+0x29E0` as the override hysteresis
band. In short: F-150 emulation proved override yield is a threshold
comparison (not a torque-cap); Transit cross-vehicle diff analysis isolated
a supervisor/gain cluster at `+0x29B8..+0x2A47`; within it, `±0.8` and `±0.5`
are paired symmetric thresholds in torque-shaped float Nm.

This keeps the inner hysteresis band tight so the rack re-engages cleanly
after transient driver input — only the outer yield threshold is relaxed.

## What each change does (plain-language)

- **Min-speed 0 m/s**: LKA engages from any speed including standstill.
- **Lockout timers zero**: no 10-second forced drop-out during continuous
  assist; no tug-then-silence pattern.
- **Torque curve 3×**: at 70 kph the rack can apply up to 3.5 Nm instead of
  1.0 Nm, enough to actually hold a lane through moderate curves.
- **Override threshold 2×**: driver torque must exceed ±1.6 Nm (instead of
  ±0.8) before the rack classifies you as "fighting the wheel" and yields
  assist. Reaction torque generated by a hard cornering maneuver stays
  inside the quiet band.

The torque-curve patch alone doesn't fix the big-turn failure, because if
the rack yields assist it never uses the extra authority. The override
patch is what closes that gap.

## Expected behavior

- Engages from low speed, holds indefinitely, drives harder through curves,
  and doesn't bail when commanded angle hits the range that previously
  caused yield.
- Hands-on detection should still behave normally — the change is in the
  override yield path, not the hands-on-wheel detector (which is a separate
  signal path upstream of the rack).

## Caveats and risk

**Medium confidence** on the override threshold being an LKA-specific
yield gate versus some neighboring supervisor feature (TJA, ESA, centering
hold). If first drive reveals unexpected behavior, back-calculate from
which symptom changed:

- If LKA now ignores real driver grabs → `±0.8` was the hands-on detector,
  revert and try `+30` at `cal+0x29DC` or `+1000` persistence counter at
  `cal+0x29E8` instead.
- If big turns still fail but LKA feels slightly different in other ways
  → this is not the right record for your symptom; move to the `+0x06AE`
  u16 timer family.
- If nothing changed → revert and hunt elsewhere.

**No Transit emulator proof.** The F-150 Unicorn harness confirmed the
override-threshold mechanism at the F-150 RAM mirror `_DAT_fef263de`, and
the Transit cal layout at `+0x29D4` pattern-matches that mechanism. But we
haven't executed Transit strategy code against these bytes in an emulator
(Unicorn-pr1918 doesn't decode all V850E2M extension opcodes). Drive test
validates.
