# 2022 Ford F-150 — PSCM

**Different platform from Transit.** F-150 uses a different PSCM vendor and a different MCU / firmware layout. Files archived here for architectural comparison only — they are **not** cross-flashable to any other vehicle in this repo.

## Identity

| Field | Value |
|---|---|
| PSCM strategy prefix | `ML3V-14D003-*` (primary) / `ML34-14D003-*` (variant) |
| EPS core prefix | `ML34-14D005-*` |
| Cal prefix | `ML34-14D007-*` |
| Cal layout | Different from Transit/Escape — **not** cross-compatible |
| Cal address | Unconfirmed (likely different base) |

## Why F-150 is here at all

Early in the project, I assumed F-150 was the obvious LCA donor for Transit because it's a bigger/fancier Ford that ships with Co-Pilot 360. It's **not** — the F-150 PSCM is a different vendor with a different cal layout. Offsets don't align; bytes at `cal + 0x06B0` on F-150 are meaningless on Transit.

**Lesson:** Before assuming cross-compatibility, diff cal sizes and look for the `TKP_INFO` vendor string. Transit/Escape share it; F-150 doesn't.

Keeping these files in the repo documents that finding and makes the mistake reproducible for others.

## What you might still learn from F-150 files

- **Cal layout comparison** — useful to see how a different vendor organizes the same kind of data.
- **V850E2M vs. whatever F-150 uses** — disassembly of `ML3V-14D003-BD` might reveal a different MCU family.
- **LCA feature implementation** — different vendor, same Ford-level LCA CAN protocol. How they wire it differs.
